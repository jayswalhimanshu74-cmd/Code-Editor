Fixed source filename causes overwrite race in concurrent runs
ExecutionWorker.processExecution — Main.java overwrite collision
Critical Worker
▾
Problem
The worker writes source code to a fixed path: {cloud-workspaces}/{roomId}/Main.java. Two concurrent executions for the same language in the same room overwrite each other. The per-room Redis lock helps but does not cover different-language simultaneous runs (different filenames). More importantly, an ephemeral sandbox for room A mounts the entire room directory, so a concurrent write to the same room dir during an active container run can inject changed files mid-execution.

Warning
▾
Problem
The sync query uses findByRoomAndParentIsNull — only top-level files and folders. The recursive writeNodeToDisk then uses findByParent(file) to walk children, which is correct. The approach is correct for normal trees but has two issues: 1. N+1 queries: each folder triggers a separate SELECT. 2.
